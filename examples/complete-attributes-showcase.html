data-next-show="shipping.cost === 0" - Show when no shipping cost

🔧 Setting Shipping Context

To use shipping attributes, you need to be within an element that has data-next-shipping-id:

<div data-next-shipping-id="1">
  <span data-next-show="shipping.isFree">FREE SHIPPING!</span>
  <span data-next-hide="shipping.isFree">
    Shipping: <span data-next-display="shipping.cost">$0.00</span>
  </span>
</div>

💡 Cart vs Shipping Context

Cart-level shipping: Use cart.shipping for the total cart shipping cost

Context shipping: Use shipping.* within a data-next-shipping-id context

Example:

<!-- Cart total shipping -->
<p>Cart shipping: <span data-next-display="cart.shipping">$0.00</span></p>
<p data-next-show="cart.shipping.raw === 0">Free shipping on your order!</p>

<!-- Package-specific shipping -->
<div data-next-shipping-id="2">
  <p>This package ships via: <span data-next-display="shipping.code">Express</span></p>
  <p data-next-show="shipping.cost > 0">
    Shipping fee: <span data-next-display="shipping.cost">$12.99</span>
  </p>
</div>

📝 Implementation Notes

🎯 Context Awareness

Package Context: Elements inside [data-next-package-id] automatically get package context

Selection Context: Use data-next-selector-id or embed ID in path: selection.{selectorId}.property

Shipping Context: Elements inside [data-next-shipping-id] can access shipping properties

Global Context: Cart and order properties work anywhere on the page

⚡ Performance Features

Auto-Updates: All displays update automatically when underlying data changes

Smart Caching: Values are cached and only updated when they actually change

Hidden Element Skip: Hidden elements skip updates for better performance

🔧 Developer Tips

Debug Mode: Add ?debug=true to URL for detailed logging

Raw Values: Use .raw suffix for unformatted numeric values

Fallbacks: Missing values display as empty strings or "N/A" on errors

CSS Classes: Elements get .display-visible and .display-hidden classes
