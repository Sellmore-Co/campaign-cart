Fix common Campaign Cart issues quickly with this comprehensive troubleshooting guide. Most problems can be resolved in minutes with the right approach.

## Quick Diagnostic Steps

<Steps>
1. **Check Browser Console** - Look for JavaScript errors (F12 → Console)
2. **Verify Installation** - Ensure Campaign Cart script loaded
3. **Test API Credentials** - Confirm your API key and campaign ID work
4. **Enable Debug Mode** - Get detailed logging information
</Steps>

<Aside type="tip">
**Debug Mode:** Add `<meta name="os-debug" content="true">` to see detailed logging in your browser console. This reveals what Campaign Cart is doing behind the scenes.
</Aside>

## Installation Issues

### ❌ Campaign Cart Not Loading

**Symptoms:**
- Nothing happens when clicking buttons
- `window.twentyNineNext` is undefined
- No console messages

**Solutions:**

<Tabs>
  <TabItem label="Check Script Loading">
    <Code 
      code={`// Open browser console and run:
console.log('Campaign Cart loaded:', typeof window.twentyNineNext !== 'undefined');

// Should output: "Campaign Cart loaded: true"`}
      lang="javascript"
    />
    
    If false, check:
    - Script URL is correct
    - No 404 errors in Network tab
    - Ad blockers aren't blocking the script
  </TabItem>
  
  <TabItem label="Network Issues">
    Common causes:
    - **Ad Blockers** - Disable temporarily to test
    - **Firewall** - Check corporate firewall settings
    - **HTTPS/HTTP** - Ensure protocol matches your site
    - **Typos** - Verify script URL spelling
  </TabItem>
  
  <TabItem label="Browser Compatibility">
    <Code 
      code={`// Check browser support
if (!window.fetch) {
  console.error('Browser too old - needs modern browser');
}

// Minimum requirements:
// Chrome 90+, Firefox 88+, Safari 14+, Edge 90+`}
      lang="javascript"
    />
  </TabItem>
</Tabs>

**Symptoms:**
- Campaign Cart loads (`window.twentyNineNext` exists)

<meta name="os-debug" content="true">`}

## API and Credentials Issues

### 🔑 Invalid API Key Errors

**Symptoms:**
- Console shows "Invalid API key" or 401 errors
- Products don't load
- Checkout fails

**Solutions:**

<CardGrid>
  <Card title="1. Verify API Key" icon="key">
    - Log in to [29next Dashboard](https://dashboard.29next.com)
    - Check API Keys section
    - Ensure key is **active** and **not expired**
    - Copy exact key (including any prefixes)
  </Card>
  
  <Card title="2. Check Key Format" icon="code">
    <Code code={`// Correct format examples:
"live_pk_1234567890abcdef"  // Live key
"test_pk_abcdef1234567890"  // Test key

// Wrong formats:
"1234567890abcdef"          // Missing prefix
"pk_1234567890abcdef"       // Missing environment`} lang="javascript" />
  </Card>
</CardGrid>

## Cart and Product Issues

### 🛒 Cart Count Not Updating

**Symptoms:**
- Clicking "Add to Cart" does nothing
- Cart count stays at 0
- No visual feedback

**Check HTML Attributes:**

<Tabs>
  <TabItem label="Common Mistakes">
    <Code 
      code={`<!-- ❌ Wrong -->
<button data-action="toggle-item" data-package="1">Add</button>
<span data-cart-count>0</span>

<!-- ✅ Correct -->
<button data-os-action="toggle-item" data-os-package="1">Add</button>
<span data-os-cart-count>0</span>`}
      lang="html"
    />
    
    **Key Points:**
    - All attributes must start with `data-os-`
    - Use exact spelling: `toggle-item`, not `toggle_item`
  </TabItem>
  
  <TabItem label="Package ID Issues">
    <Code 
      code={`<!-- Check package ID exists -->
<button data-os-action="toggle-item" data-os-package="999">
  This won't work if package 999 doesn't exist
</button>

<!-- Use product profiles instead -->
<button data-os-action="toggle-item" data-os-profile="starter-kit">
  More reliable with profiles
</button>`}
      lang="html"
    />
  </TabItem>
</Tabs>

### 💰 Prices Show $0.00

**Symptoms:**
- All products show $0.00
- Cart total is always $0.00
- Package prices don't load

**Solutions:**

<Steps>
1. **Verify Package IDs**
   - Check your 29next dashboard for correct package IDs
   - Ensure packages are active and published

2. **Check Campaign Status**
   - Campaign must be active
   - Payment methods must be configured
   - Tax settings properly configured

3. **Test with Product Profiles**
   
   <Code code={`<!-- Instead of package IDs -->
   <span data-os-profile-price data-os-profile-id="starter-kit">Loading...</span>`} lang="html" />
</Steps>

## Performance Issues

### 🐌 Slow Loading

**Symptoms:**
- Cart takes seconds to respond
- Checkout form loads slowly
- Price updates are delayed

**Optimization Steps:**

<Steps>
1. **Use CDN Version**
   <Code code={`<script src="https://rtc2.29next.com/campaign-cart/29next.min.js"></script>`} lang="html" />

2. **Optimize Images**
   - Compress product images
   - Use appropriate formats (WebP, AVIF)
   - Implement lazy loading

3. **Reduce DOM Complexity**
   - Minimize HTML elements
   - Remove unused CSS
   - Optimize JavaScript
</Steps>

## Getting Additional Help

### Before Contacting Support

<Steps>
1. **Gather Information**
   - Browser and version
   - Console error messages
   - Steps to reproduce
   - Screenshots if helpful

2. **Test in Incognito Mode**
   - Rules out browser extensions
   - Clears cache issues
   - Isolates the problem

3. **Try Different Browser**
   - Tests browser compatibility
   - Confirms if issue is global
</Steps>
